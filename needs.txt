# Glass Factory — Phase 0 Manual MVP

## Commercial flow

Quotation -> Sales Order -> Delivery Note -> Sales Invoice

Production flow:

Sales Order -> Cutting Job -> Stock Entry Repack #1 -> Glass Processing Job -> Stock Entry Repack #2 -> Delivery Note -> Sales Invoice

The exact final processed glass Item is created or reused before Sales Order submission and remains the same Item on all commercial documents.

## Selling model

- Mark glass rows with `gf_is_glass_item` on Quotation Item / Sales Order Item.
- Enter raw sheet Item, dimensions, thickness, and processing flags on the row.
- `resolve_glass_items` creates/reuses raw sheet, cut WIP, and final Items deterministically.
- `item_code` on selling rows is always the exact final Item.

Example Item codes:

- Raw Sheet: `GLS-CLEAR-8MM-3210X2250`
- Cut WIP: `GLS-CLEAR-8MM-1200X800-CUT`
- Final: `GLS-CLEAR-8MM-1200X800-POL-HOL-TMP`

## Master data

Configure **Glass Factory Settings**:

- Warehouses: raw, cut WIP, final goods, remnants, scrap
- Item groups for each glass role
- Default UOM, min remnant/chargeable area rules
- `enable_cop` (off by default in Phase 0)

## Cutting Job (manual)

1. Link one or more submitted Sales Orders.
2. Pull pieces from Sales Orders.
3. Add source sheets (raw sheet or remnant).
4. Create and submit Repack #1: Raw/Remnant -> Cut WIP + Remnant + Scrap.
5. Optionally complete the Cutting Job, then create a Glass Processing Job.

## Glass Processing Job (manual)

1. Inputs are Cut WIP Items from the linked Cutting Job.
2. Outputs are exact final Sales Order Items.
3. Create and submit Repack #2: Cut WIP -> Final.
4. Complete the job.

## Delivery

- Create Delivery Note from Sales Order.
- Deliver only the final Item linked on the Sales Order row.
- Delivered quantity cannot exceed `gf_processed_qty` from Repack #2.

## Exclusions

Phase 0 simple glass does not use BOM, Work Order, Production Plan, or Job Card.

## COP (later / optional)

COP file generation and optimization remain dormant behind `enable_cop`. Manual entry must work without COP.

Legacy DocTypes (`Glass Cut Piece`, `Cutting Job Linked SO`, `Cutting Job Tabular File`, `Glass Cutting Settings`) are removed after migrate.
